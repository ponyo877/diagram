# 新規Webアプリ開発要望
astahのクラス図描画の機能のみがfigmaのように極めて直感的に描け、かつ描いたクラス図をplantUMLなどの既存の構造化されたテキストで出力できるWebアプリを作りたいです。 クラス図を書く際にfigmaのように一つのスペースに複数のユーザが同時にアクセスでき、編集内容が互いに同期されながら進める機能を持ちたいです。
astahのクラス図描画はこちら(https://astah.change-vision.com/ja/manual/423-class-diagram.html)です。

画面構成と可能な操作
- ランディングページ: 
  - ログインなどの機能は一旦不要で「クラス図を作成」というアプリタイトルとロゴとボタンだけあるシンプルなページ
  - 「クラス図を作成」を押すとクラス図作成ページに遷移する
- クラス図作成ページ:
  - ランディングページで「クラス図を作成」を押すたびに毎回新規に作成されるページ
  - URLのpathの末尾がuuidになっており、クラス図作成ページのURL全体を知っている人だけが同じクラス図作成ページにアクセスできる
  - 複数のユーザが同時にアクセスでき、アクセスしているユーザの名前とそのユーザのアイコンの現在位置がユーザごとに違う色でユーザ名付きの矢印として表示されている
  - 全てのユーザはastahのクラス図描画をfigmaのような直感的な以下のような操作で作成することができる
    - なんのオブジェクト(クラス, 列挙型, パッケージ, サブシステム, エンティティ, バウンダリ, コントロールなど)を作るかを画面下部で選択できる
    - オブジェクトを作った後にオブジェクトの位置やタイトルや枠の大きさや色の変更、属性の追加、オブジェクトの種類の変更、オブジェクトとオブジェクトの関連付けを行うことができる
    - オブジェクトとオブジェクトをまずシンプルな線で繋ぐことができ、繋いだ後にその線がどんな種類の関係(汎化, 実現, 依存など)か変更することができる
    - スペースにあるクラス図をplantUMLとしてエクスポートすることができる
  - クラス図を作成に入ったタイミングでは全てnonameというユーザ名だが、後からを変更することができる
  - スペースの変更内容は常に自動保存される
